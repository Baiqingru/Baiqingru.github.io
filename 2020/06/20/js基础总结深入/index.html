JavaScript基础总结(以问题为导向)

1.基础原理部分

1.计算机的组成是怎样的 ?

  • 硬件
    • 输入设备 - 键盘等
    • 输出设备 - 显示器等
    • 硬盘 - 固定数据的存储
      • ==为什么不用它存储直接操作数据数据?== => 因为硬盘读取时候速度很慢,造成了cpu的闲置,这样的话,效率很低下。
    • 内存 - 临时数据存储
      • ==为什么要内存?== => 因为硬盘读取数据的操作较慢,为了更高的效率,就使用内存,内存虽然不大,但是内存的读取数据快速
      • ==内存是啥?== =>内存条通电之后产生的一个存储数据的临时空间
        • ==内存能再分类吗?==
          • 堆内存 - 放的是对象,就是指向栈那边存储它 “ 门牌号 “ 的内存
            • ==为什么对象放在堆里面?==
              • 因为对象的大小会发生变化,若放在栈中,会影响变量的读写速度,地址大小是恒定的
          • 栈内存 - 全局变量 & 局部变量
            • ==变量是什么?== => 可变化的量: 变量名+ 变量值 - 每个变量对应一个小内存 (内存的标识)
              • 变量名 –> 查找对应的内存
              • 变量值 –> 内存中保存的数据
        • ==内存数据类型分为哪几类?==
          • 地址数据
          • 普通数据
      • ==内存操作的是什么?===> 数据
        • ==啥是数据?== => 表示信息的载体,其本质是 0111…..
        • ==数据能干什么?==
          • 可以传递信息
          • 可以进行运算 => ==哪些运算?==
            • 算数 & 逻辑 & 赋值 & 运行函数
        • ==数据的基本特性==
          • 可读取
          • 可传递
        • 一切都是数据, 程序操作的最终都是在操作数据
      • ==内存是如何分配释放空间的?==
        • 分配内存
          • 声明变量、函数或创建对象时, JS引擎会自动为此分配一定大小的内存来存放对应的数据
        • 释放内存
          • 清空内存中的数据, 标识内存可以再分配使用(内存不释放就不能复用)
          • ==自动释放与垃圾回调器回调的差异?==
            • 自动释放是js引擎的一种自动的行为
              • 为执行函数分配的栈空间内存: 函数执行完自动释放
            • 垃圾对象回收需要的是垃圾回收器
              • 存储对象的堆空间内存: 当内存没有引用指向时, 对象成为垃圾对象, 垃圾回收器后面就会回收释放此内存
              • let 变量 = null
      • ==内存的生命周期是什么样的 ?==
        • 内存条->通电->内存空间产生 -> 存储-> 数据处理-> 断电->内存空间和数据均消失
      • 内存的
    • CPU - 处理数据和计算数据
  • 软件
    • 系统软件
    • 应用软件

2. JS的数据类型

2.1 分类

  • 基本数据类型
    • String
    • number
    • Boolean : false/true
    • null
    • undefined
  • 引用数据类型
    • Function
    • Array
    • Object

2.2 判断

  • typeof
    • 不可判断
      • null & Object
      • Array & Object
    • 其余均可判断
  • instanceof
    • 专用: 用以判断对象的具体类型
  • ===
    • 仅可判断
      • null
      • undefined

2.3 思考题

  • undefined与null的区别?
    • undefined 定义未赋值
    • null 定义且赋值为null
  • 何时将变量赋值为null?
    • 初始赋值 :
      • 占位, 表明此变量未来用于存放对象类型
    • 对象”寿终” :
      • 使对象变成垃圾对象, 等待垃圾回收器回收
  • 变量类型与数据类型的区别?
    • 数据的类型
      • 基本类型
      • 对象类型
    • 变量的类型 (变量内存值的类型)
      • 基本类型: 保存的是基本类型的数据
      • 引用类型: 保存的是地址值

4.函数基础

4.1 函数的基础思考

  • 什么是函数?

    • 实现特定功能的n条语句的封装体
    • 只有函数是可以执行的, 其它类型的数据不能执行

  • 为什么要用函数?

    • 提高代码复用
    • 便于阅读交流

  • 如何定义函数?

    • 函数声明
    • 表达式

  • 如何调用(执行)函数?

    • test(): 直接调用
    • obj.test(): 通过对象调用
    • new test(): new调用
    • ==test.call / apply(obj): 临时让test成为obj的方法进行调用==

4.2 回调函数

  • 什么函数才是回调函数?

    1). 你定义的

    2). 你没有调

    3). 但最终它执行了(在某个时刻或某个条件下)

  • 常见的回调函数?

    • Dom事件回调函数 ==>发生事件的Dom元素
    • 定时器回调函数 ===>window
    • Ajax请求回调函数
    • 生命周期回调函数

4.3 IIFE (Immediately-Invoked Function Expression)

  • IIFE是什么?

    • 立即调用的函数表达式，也就是说，声明函数的同时立即调用这个函数。

  • 为什么使用 IIFE?

    • IIFE的出现是为了弥补JS在scope方面的缺陷：

      • JS只有全局作用域（global scope）、函数作用域（function scope），从ES6开始才有块级作用域（block scope）

对比现在流行的其他OOP语言可以看出，JS在访问控制方面很不友好！如何实现作用域的隔离呢？在JS中，只有function才能实现作用域隔离，故要将一段代码中的变量、函数等的定义隔离出来，只能将这段代码封装到一个函数中。
在我们通常的理解中，将代码封装到函数中的目的是为了复用。

在JS中声明函数的目的在大多数情况下也是为了复用，但是JS迫于作用域控制手段的贫乏，我们也经常看到只使用一次的函数：这通常的目的是为了隔离作用域了！既然只使用一次，那么立即执行好了！既然只使用一次，函数的名字也省掉了！这就是IIFE的由来。

  • 作用
    • 隐藏实现
    • 不会污染全局命名空间
    • JS模块化

4.4 函数的this指向问题

  • this是什么?
    • 任何函数本质上都是通过某个对象来调用的,如果没有直接指定就是window
    • 所有函数内部都有一个变量this
    • 它的值是调用函数的当前对象
  • 如何确定this的值?
    • test(): window
    • p.test(): p
    • new test(): 新创建的实例对象
    • p.call(obj): obj
